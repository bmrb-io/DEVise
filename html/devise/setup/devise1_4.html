Architectures

DEVise 1.5 is currently available for the following architectures:

  • Intel/Solaris
  • SPARC/Solaris

Setup Procedure

If you do decide to run DEVise 1.5, here's how to go about it.

  • Go to the directory in which you want to have your DEVise files (a 'devise' directory will be created to hold them).
  • Run the script /p/devise/public/set_up_user.

To run DEVise, go to the devise/run directory that the set_up_user script created, and run devise or devisepub.

devise runs DEVise with your own sessions, schemas, and data sources (example files are installed in your devise directory).

devisepub runs DEVise with the public sessions, schemas, and data sources. (If you are not in the miron:devise AFS group, you won't have write permissions on any of those files.)

Note that a number of paths in your own DEVise files are relative paths. This means that when you run devise, you must run it from the devise/run directory unless you change all of the paths to absolute paths. You can run devisepub from anywhere.

Transitioning from 1.3

If you already have sessions, schemas, and data that you have used in DEVise 1.3, you can transition them to DEVise 1.5. Sessions, schemas, and data files can be used without changes, but the data source catalog must be changed.

To translate your 1.3 data source catalog to a 1.5 data source catalog, follow this procedure:

  • Go to the directory where your 1.3 data source catalog (sourcedef.tcl) is.
  • Run the command "/p/devise/public/transcat.tcl > <temp file>".
  • Edit <temp file> and remove the first two lines.
  • Go to your devise/schema/schema directory.
  • Copy catalog.dte (the DEVise 1.5 data source catalog file) to catalog.dte.bak (it's very easy to corrupt the data catalog in such a way that it won't work anymore).
  • Edit catalog.dte and append <temp file> to it.

Transitioning from 1.4

At this point (July 8, 1998), you shouldn't have to do anything special to transition from DEVise 1.4 to DEVise 1.5. You will notice that the colors in your sessions have changed because of the new color manager, but they should still work -- you may want to change them for better appearance, though.

Devise-sup Mailing List

If you set yourself up to run DEVise 1.5, you should subscribe to the devise-sup (DEVise support) mailing list. This is a mailing list intended to provide information to DEVise users and to allow them to pose questions about using DEVise. Subscribing to this mailing list will allow you to get information such notices about alpha releases of a new version of DEVise.

To subscribe to devise-sup, send email to devise-sup-request@cs.wisc.edu with the following as the message body: 

	subscribe

Send mail to the DEVise User Support Hotline.
